### 🏆 Рекомендуемый инструмент: **Cline**

**Аргументы:**

1. **Локальные файлы** — расширение для VS Code, прямой доступ к файловой системе проекта (**4/5**).
2. **Безопасность** — полная обработка на локальной машине, можно подключать локальные модели (**5/5**).
3. **Бюджет** — бесплатный, оплата только за API-токены (**5/5**).
4. **Сложность** — справляется с проектами средней сложности, код чистый и структурированный (**4/5**).
5. **Выгрузка на хостинг** — проект остаётся в локальной папке, любой файл можно выгрузить на хостинг.

### 🥈 Альтернатива: **Aider**

Если Cline не подойдёт, **Aider** — отличная замена:
- Бесплатный, open-source.
- Прямой доступ к файловой системе.
- Каждое изменение через Git — удобная история версий.
- Код работает без правок в 71% случаев.

## 6. Практическое сравнение: Koda vs Cursor

Для проверки выбранных инструментов был создан одинаковый проект — **сайт онлайн-записи к парикмахеру и массажисту**. Ниже представлено сравнение результатов работы двух AI-агентов.

### 6.1. Проект, сгенерированный Koda (папка `prg1`)

**Структура проекта:**
```
prg1/
├── index.html      (583 строки)
├── css/
│   └── style.css   (1233 строки)
└── js/
    └── script.js   (646 строк)
```

**Реализованные функции:**
- ✅ Полноценная многосекционная структура (Hero, Преимущества, Услуги, Мастера, Онлайн-запись, Отзывы, Контакты, Футер)
- ✅ Табы переключения услуг (парикмахерские / массаж)
- ✅ Кастомный календарь на чистом JavaScript с навигацией по месяцам
- ✅ Генерация временных слотов с эмуляцией «занятых» часов
- ✅ Динамическое обновление списка услуг при выборе типа
- ✅ Маска ввода телефона (+7 формат)
- ✅ Полная валидация формы с отображением ошибок
- ✅ Слайдер отзывов с автопрокруткой и точками навигации
- ✅ Intersection Observer — анимация появления блоков при скролле
- ✅ Гамбургер-меню для мобильных устройств
- ✅ Кнопка «Наверх» с плавной прокруткой
- ✅ Плавный скролл по якорным ссылкам
- ✅ SEO meta-теги (description, keywords, Open Graph)
- ✅ SVG-иконки для преимуществ, контактов и соцсетей
- ✅ Встроенная карта OpenStreetMap
- ✅ Адаптивность: 1024px (планшет), 768px (мобильный), 480px (малые экраны)

**Приложенные скриншоты (Koda):**
| Скриншот | Описание |
|---|---|
| `ScreenShot\koda.png` | Главный экран (Hero) — полный первый экран с фоновым изображением |

- Подробные комментарии к каждому блоку кода

### 6.2. Проект, сгенерированный Cursor (папка `prg2`)

**Структура проекта:**
```
prg2/
└── index.html (1565+ строк — единый файл со встроенными CSS и JS)
```

**Реализованные функции:**
- ✅ Полноценная многосекционная структура (аналогично Koda)
- ✅ Табы переключения услуг
- ✅ Кастомный календарь с dropdown-открытием
- ✅ Генерация временных слотов с рандомными «занятыми» часами
- ✅ Динамическое обновление списка услуг
- ✅ Маска ввода телефона (+7 формат)
- ✅ Полная валидация формы с отображением ошибок
- ✅ Слайдер отзывов с точками навигации и стрелками
- ✅ Intersection Observer — анимация появления блоков
- ✅ Гамбургер-меню для мобильных устройств
- ✅ Кнопка «Наверх»
- ✅ Модальное окно успешной записи
- ✅ ARIA-атрибуты для доступности (aria-label, aria-expanded, role)
- ✅ Динамический рендеринг карточек услуг и мастеров через JavaScript
- ✅ Адаптивность через media queries (576px, 768px, 992px)

**Приложенные скриншоты (Cursor):**
| Скриншот | Описание |
|---|---|
| `ScreenShot\cursor.png` | Главный экран (Hero) — полный первый экран с фоновым изображением |

**Особенности реализации:**
- Всё в одном файле (HTML + CSS + JS) — удобно для быстрого деплоя, но менее удобно для поддержки
- CSS встроены в `<style>` внутри `<head>`
- JavaScript встроен в `<script>` перед закрывающим `</body>`
- Карточки услуг и мастеров рендерятся динамически через JS (data-driven подход)
- Модальное окно вместо inline-сообщения об успехе
- Более компактный код за счёт динамической генерации HTML

### 6.4. Выводы

#### Koda — что понравилось:
1. **Чистая архитектура** — разделение на HTML/CSS/JS соответствует лучшим практикам веб-разработки
2. **Подробные комментарии** — каждый блок кода описан, что облегчает понимание и доработку
3. **Premium-дизайн** — продуманная цветовая палитра, плавные анимации, качественные SVG-иконки
4. **SEO-оптимизация** — правильные meta-теги, Open Graph, семантическая разметка
5. **Готовность к хостингу** — стандартная структура файлов готова к загрузке на любой хостинг

#### Koda — что можно улучшить:
- Добавить ARIA-атрибуты для повышения доступности
- Реализовать динамический рендеринг карточек (data-driven подход)
- Добавить модальное окно вместо inline-сообщения об успехе

#### Cursor — что понравилось:
1. **Data-driven подход** — карточки генерируются из данных, а не прописаны статически в HTML
2. **Доступность (a11y)** — ARIA-атрибуты, семантическая навигация
3. **Компактность** — всё в одном файле, удобно для быстрого прототипа
4. **Модальное окно** — более элегантное решение для подтверждения записи
5. **Динамическая генерация** — код короче, легче масштабировать данные

#### Cursor — что можно улучшить:
- Разделить код на отдельные файлы для лучшей поддержки
- Добавить SEO meta-теги и Open Graph
- Добавить более подробные комментарии к коду
- Улучшить визуальный дизайн (анимации, тени, градиенты)
